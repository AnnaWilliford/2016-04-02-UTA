Some supplemental notes about commands:
  • cut:
    • When using the '-d' option to specify a delimiter, you are limited to single-character delimiters. This works great for most files, but it may not if you are trying to do more complicated cuts
  • uniq:
    • Remember that uniq only collapses successive terms. This means it will only operate "correctly" if you have the same text string on successive lines. This is why you must sort first!
  • >/<:
    • You are seeing how '>' is used to write output into a file. You can also use '<' to read a file into a command/program. We won't get into this today, but this is hopefully intuitive and can be powerful.

Very advanced:
  • Someone asked how one would specify the column name (i.e., from the header) rather than the column index (i.e., the number of the column) when sorting or cutting. There is no easy way in one command, but you can run a command to first determine the column index of your desired column name:
  • head -1 OECD_Countries_Full.txt | tr -s '\t' '\n' | nl -nln |  grep "Value" | cut -f1
  • This will return the column number of the "Value" column.
  • See http://stackoverflow.com/questions/32612859/bash-retrieve-column-number-from-column-name

import pandas as pd
import sqlite3
import matplotlib.pyplot as plt

%matplotlib inline

query = "SELECT D.County, D.Poverty, M.Infant_Mortality \
                       FROM Demographics AS D \
                       JOIN MeasuresOfBirthAndDeath AS M ON \
                       D.State = M.State AND D.County = M.County;"

con = sqlite3.connect("chsi.sqlite")
df = pd.read_sql_query(query, con)

df.plot(kind='scatter', x='Poverty', y='Infant_Mortality')

a = df.plot(kind='scatter', x='Poverty', y='Infant_Mortality')
